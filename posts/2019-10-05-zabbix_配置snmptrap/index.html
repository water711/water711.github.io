Zabbix_配置snmptrap监控交换机

Oct 5, 2019
2 min read

一、安装配置snmptrapd

[root@zabbix1 ~]# yum install -y net-snmp net-snmp-utils net-snmp-perl [root@zabbix1 ~]# vim /etc/snmp/snmptrapd.conf authCommunity log,execute,net public #设置团体名 traphandle default /usr/sbin/snmptthandler #设置snmptt为trap接收器，此时还没安装snmptt

[root@zabbix1 ~]# systemctl start snmptrapd [root@zabbix1 ~]# systemctl enable snmptrapd

#防火墙开放snmp trap端口 [root@zabbix1 ~]# firewall-cmd –permanent –add-port=162/udp [root@zabbix1 ~]# firewall-cmd –reload

#测试 设备打开snmptrap，设置接收主机IP和团体名,手动触发告警,查看是否接收到告警信息 [root@zabbix1 ~]# tail -F /var/log/messages

 

二、更改snmptrapd日志路径

snmptrapd接收到的log默认会到/var/log/messages里面，但如果snmptrapd设备较多，会产生大量日志，会干扰我们平时查看系统日志信息，所以我们要把snmptrapd的日志信息单独存放

-L：日志选项。支持的输出方式有标准错误（e），标准输出（o），文件（f），syslog（s）。

-Ls：输出到系统日志文件

-Lf：输出到指定文件

#新建日志文件存放目录 [root@zabbix1 ~]# mkdir /var/log/snmptrap/

#修改systemd服务文件 [root@zabbix1 ~]# vim /usr/lib/systemd/system/snmptrapd.service

[Service] Type=notify #Environment=OPTIONS="-Lsd" #注释该行 EnvironmentFile=-/etc/sysconfig/snmptrapd #ExecStart=/usr/sbin/snmptrapd $OPTIONS -f
ExecStart=/usr/sbin/snmptrapd -Lf /var/log/snmptrap/snmptrap.log -f #指定snmptrapd日志存放位置 ExecReload=/bin/kill -HUP $MAINPID

 

[root@zabbix1 ~]# systemctl daemon-reload #修改systemd文件后，要重新加载 [root@zabbix1 ~]# systemctl restart snmptrapd.service

 

三、安装配置snmptt

[root@zabbix1 ~]# yum -y install epel-release [root@zabbix1 ~]# yum -y install perl-Sys-Syslog snmptt //snmptt需要Perl Syslog module支持

[root@zabbix1 ~]# vim /etc/snmp/snmptt.ini date_time_format= %Y/%m/%d %H:%M:%S net_snmp_perl_enable = 1 translate_log_trap_oid = 2 （0：显示数字OID，1：显示OID名称，2：显示OID所属模块名及其名称） log_file = /var/log/snmptt/snmptt.log（默认即可）

 

自定义trap格式化规则

[root@zabbix1 ~]# cp /etc/snmp/snmptt.conf{ , .bak} #备份配置文件 [root@zabbix1 ~]# vim /etc/snmp/snmptt.conf EVENT Linkdown .1.3.6.1.6.3.1.1.5.3 “Status Events” Warning FORMAT ZBXTRAP $aA LinkDown on interface $4. Admin state: $2. Operational state: $3

EVENT Linkup .1.3.6.1.6.3.1.1.5.4 “Status Events” Normal FORMAT ZBXTRAP $aA Linkup on interface $4. Admin state: $2. Operational state: $3

EVENT general .* “General event” Unknown FORMAT ZBXTRAP $aA $1

[root@zabbix1 ~]# systemctl restart snmptt [root@zabbix1 ~]# tail -F /var/log/snmptt/snmptt.log

 

四、Zabbix上配置SNMP监控

[root@zabbix1 ~]# vim /etc/zabbix/zabbix_server.conf StartSNMPTrapper=1 #Zabbix开启SNMPTrapper SNMPTrapperFile=/var/log/snmptt/snmptt.log #修改SNMPTrapper文件路径

[root@zabbix1 ~]# systemctl restart zabbix-server
